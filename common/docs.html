Описание работы основных модулей программы

Графематический анализ

Модуль графематического анализатора выполняет токенизацию входного текста.
На начальном этапе анализа каждому токену присваивается один из графематических дескрипторов:

Дескриптор Описание
- DEL # Delimiter (разделитель)
- SIG # Signum (знак препинания/пунктуации)
- RLE # Russian Lexeme (русская лексема)
- LLE # Latin Lexeme (лексема на латинице)
- DC # Digit Complex (цифровой комплекс)
- END # End of sentence (конец предложения)
- COMPOSITE # Composite gprapheme (составная графема)
- SYM # Another single symbols (остальные одиночные символы)

Следующий этап анализа присваивает каждому токену одну или несколько меток:

Дескриптор Описание
- WORD # Word (слово)
- NUMBER # Number (число)
- SPACE # Space (пробельные символы)
- LINK # Link (ссылка)
- EMAIL # Email (электронная почта)
- HASHTAG # Hashtag (хештег)
- CYRIL # Cyril (кирилица)
- LATIN # Latin (латиница)
- PUNCT # Punctuation marks (знаки пунктуации)
- MARKUP # Markup symbols (символы разметки)
- QUOTE # Quote marks (символы цитирования)
- OPENING # Opening symbol (открывающий символ)
- CLOSING # Cloing symbol (закрывающий симол)
- OTHER # All other symbols (все остальные символы)

Для того, чтобы увидеть на экране метку, сопоставленную конкретному токену, достаточно
навести на него указатель мыши.


Морфологический анализ

Модуль морфологического анализатора производит анализ токенов, полученных в процессе работы
грамматического анализатора. Каждому токену в результате анализа присваивается метка, содержащая морфологическую характеристику
токена, например: Часть речи, род, число, падеж (и другие характеристики, присущие различным частям
речи.

Для того, чтобы увидеть на экране метку, сопоставленную конкретному токену, достаточно
навести на него указатель мыши.


Статистический анализ

Модуль статистического анализатора выполняет классификацию текста с помощью формулы TF-IDF.
Результатом классификации становится присваивание тексту одной из следующих категорий:

- POLITICS
- CULTURE
- SPORT
- HEALTH
- TECH
- ECONOMICS
- INCIDENT
- AUTO
- WOMAN
- ADVERTISING
- SOCIAL
- REALTY
- SCIENCE

Второй этап анализа заключается в сборе семантического ядра - выборке 10 слов, встречающихся в тексте, проранжированных по частоте вхождения в исходный текс, а также в выводе таблицы всех слов и их частоты, встречающихся в тексте. (за исключением стоп-слов)

Синтаксический анализ

Модуль синтаксического анализатора выполняет построение синтаксического дерева для каждого предложения.
Подробный алгоритм работы анализатора - Solarix

Результатом работы solarix-парсера является xml-документ, хранящий информацию по каждому предложению в виде списка узлов и связей между ними (каждый узел хранит ссылку на родительский узел и информацию о типе связи с родительским узлом).

Далее выполняется разбор xml-дерева и его перевод во внутреннюю структуру данных приложения.
С помощью библиотеки graphviz выполняется визуализация дерева для каждого предложения.
Визуализированное дерево можно сохранить в svg-файл.
Для этого нужно кликнуть правой кнопкой мыши на соответствующее предложение и в контекстном меню выбрать пункт "Сохранить как изображение".



Семантический анализ

На вход модулю семантического анализатора подаётся дерево разбора, полученное после работы синтаксического анализатора.

Из множества узлов выбираются те, тип связи между которыми принадлежит отношениям :
Объект - Субъект, Объект - Свойство, Субъект - Свойство. Данное ограничение по количеству рассматриваемых типов связей обусловлено поставленной задачей - построением онтологии на основе текста на естественном языке.

Далее полученная модель семантического графа визуализируется. Пользователю предоставляется возможность создавать, удалять узлы графа (далее - концепты), а так же добавлять, удалять и редактировать отношения между узлами.

В рамках поставленной задачи при работе с семантическим графом введены следующие обозначения:
- Концепт (понятие, объект) - узел сематнического графа
- Отношение (слово-ссылка, тип связи) - тип семантической связи между концептами

В связи с зависимостью числа ошибок и противоречий от правильности установления отношений между концептами, необходимо использовать универсальные отношения, встречающиеся в большинстве предметных областей:
- Родо-видовые (A-kind-of, "элемент - множество" и т.д.)
- Родо-видовые конкретизирующие ("класс - подкласс"
- Связи типа "часть - целое"
- Атрибутивные связи ("иметь свойство", "иметь значение")
- Функциональные связи ("иметь экземпляр", "иметь в качестве примера", "как образец")

При выборе экспорта в web-онтологию запускается алгоритм, определяющий связи между концептами и формирующий на основе связей owl-код.
Для определения некоторых видов связей используется WordNet - электронный тезаурус. WordNet работает с английским языком, поэтому для перевода концептов на английский язык используется Яндекс.Переводчик


В случае, если программе не удаётся распознать некоторые отношения, пользователь может сделать это вручную, либо пропустить этот пункт.
Данное диалоговое окно оснащено подсказкой, содержащей информацию (с примерами) по каждому типу связей, используемому в алгоритме.



Инструменты

Вкладка "Инструменты" содержит:
- Редактор графов (тот же функционал, что и в пункте семантического анализа).

- Лингвистический анализатор (функционал морфологического анализатора для одного слова)

- Классификатор текста (упрощённый функционал статистического анализатора с возможностью переобучения - пользователь вправе изменить категорию, определённую классификатором, после чего класификатор обновляет данные)
